# 学生任务书

## 任务标题

为“课间 3 分钟短休息”设计一个浏览器微型游戏，并做出一个简单能玩的网页原型。

## 今天要做什么

某职业院校希望在课程网页里加入一组“课间 3 分钟微型游戏”，让学生在课间短暂放松一下。游戏不需要下载安装，打开网页就能玩；规则要容易理解，单局时间要短，最好玩完一轮还愿意再试一次。

你今天需要根据给出的用户访谈材料，完成一个小游戏设计，并用 Codex 帮你做出一个低保真原型。

低保真原型的意思是：不用精美画面，不用完整功能，只要能在浏览器里打开，别人能试玩一两轮，就可以。

## 时间

正式任务时间为 90 分钟。  
请尽量在 90 分钟内完成五个部分：

1. 用户需求分析
2. 设计目标确定
3. 创意方案生成与选择
4. 低保真原型制作
5. 试玩反馈与修正计划

你不需要在 Codex 里专门给每个阶段打标签。阶段划分主要由研究者根据录屏、阶段输出表和 Codex 对话记录整理。如果你自然地说出“我现在开始做原型”，也可以，但忘记说也没有关系。

## 你可以使用的工具

你可以使用 Codex。你不需要自己从零写代码，但需要告诉 Codex 你想做什么，并检查它做出来的内容是否符合你的想法。

你可以问 Codex，也可以自己先想；你可以接受 Codex 的建议，也可以修改或拒绝它。重要的是，请尽量把你的判断过程说出来。

## 开始和结束时请说一句话

为了方便后面整理你的 Codex 对话记录，请你在 Codex 里用自然语言标记开始和结束。

开始时可以这样说：

```text
你好，我的编号是 P01，现在我要开始今天的任务了。请先阅读当前项目文件夹，告诉我里面有哪些文件。
```

结束时可以这样说：

```text
今天的任务到此为止。请不要再修改文件，请帮我简单总结一下你今天帮我做过哪些事情。
```

请把 `P01` 换成你的参与者编号。你不需要记任何技术指令，只要按自然语言说明即可。

## 技术要求

- 只做桌面浏览器网页。
- 只使用 HTML、CSS、JavaScript。
- 只保留一个主要玩法。
- 不做登录、数据库、排行榜。
- 不要求高保真美术，可以使用文字、色块、简单图形。
- 首次打开后，应尽量让玩家很快知道怎么玩。

## 你需要提交什么

任务结束时，请保留整个项目文件夹。里面应该包含：

- 填写完成的阶段输出表。
- 可以本地打开的网页原型。
- 试玩反馈与下一轮修改计划。

请记住：今天关注的是你的设计过程，不是作品是否精美。

## 如果你第一次使用 Codex

你不需要先学代码，也不需要先看懂网页文件。你可以直接用自然语言告诉 Codex 你现在要做什么。

例如：

```text
我现在要做用户需求分析，请帮我一起阅读访谈材料。
```

或者：

```text
我已经选好了创意方向，请帮我把它做成一个简单能玩的网页原型。
```

Codex 可能会和你讨论，也可能会帮你修改文件。你只需要继续检查：它做出来的东西是不是符合你的设计想法。

如果你请 Codex 帮你制作网页原型，它会把相关内容保存在项目文件夹里的 `网页原型` 文件夹中。你不需要自己手动编辑这些技术文件，只需要继续和 Codex 讨论游戏应该怎么做、哪里需要调整。
