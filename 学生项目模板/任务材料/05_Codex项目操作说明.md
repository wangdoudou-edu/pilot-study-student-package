# Codex 项目操作卡

今天你会在一个独立项目文件夹里完成任务。你可以直接用自然语言和 Codex 沟通，不需要会代码，也不需要记特殊指令。

## 1. 项目文件夹里有什么

```text
P01_pilot_project/
├── 任务材料/             # 任务书、用户访谈材料、阶段输出表
├── 网页原型/             # 你要做出来的小游戏网页
├── Codex对话记录/        # 研究者后续整理 Codex 对话
└── README.md
```

你主要会看到：

- `任务材料`：阅读任务、查看访谈材料、填写阶段输出表。
- `网页原型`：保存你最后做出来的网页小游戏。你不需要一开始看懂里面的文件。只要你和 Codex 讨论游戏怎么做，并请它帮你制作网页原型，相关文件会保存在这里。
- `Codex对话记录`：这个文件夹主要给研究者后续整理数据用，你不用操作。

## 2. 打开 Codex

研究者会帮你确认 Codex 已经在这个项目文件夹中打开。你只需要确认自己看到的是本次任务的项目文件夹。

## 3. 任务开始时的第一句话

请在第一句话里写上你的参与者编号，并用自然语言说明任务开始。例如：

```text
你好，我的编号是 P01，现在我要开始今天的任务了。请先阅读当前项目文件夹，告诉我里面有哪些文件。
```

任务结束时，请也用自然语言说一句：

```text
今天的任务到此为止。请不要再修改文件，请帮我简单总结一下你今天帮我做过哪些事情。
```

## 4. 使用时注意

- 可以用自然语言告诉 Codex 你想做什么。
- 可以让 Codex 帮你阅读材料、整理想法、制作网页原型、检查问题。
- 当你请 Codex 帮你制作或修改网页原型时，它会把改动保存到 `网页原型` 文件夹里。你不需要自己手动编辑这些技术文件。
- Codex 做完后，请你自己看一看结果是否符合你的想法。
- 请不要删除项目文件夹里的材料。

今天不要求你写完美提示词。你可以按照自己平时使用 Codex 的方式完成任务。

## 5. 如果你不知道怎么说

可以直接说你的真实想法。例如：

```text
我现在要根据访谈材料做用户需求分析，但我不知道怎么开始。请帮我一起看材料。
```

或者：

```text
我想做一个适合课间 3 分钟玩的小游戏，请帮我把现在的想法做成一个简单网页原型。
```

Codex 可能会修改 `网页原型` 里的文件，也可能先和你讨论方案。你只需要根据自己的判断继续推进。
